# estimator-api â€” handoff notes

Context for picking this work back up. Last updated 2026-05-30.

## What this is

A standalone **FastAPI** service that detects tax-depreciable **renovations**
from property photos, split out of the main Flask `rpdata-ai-server` (`../app.py`).
It is a thin wrapper around an OpenAI **vision** call: given an `rpId`, fetch the
property's photos (calc.duo.tax) + the renovation-items catalog (megamind), send
prompt + dataset + photos, format the model's JSON back out. It does **not**
compute construction costs (the prompt forbids it).

Lives in `estimator-api/`. Its own venv is `venv/` (NOT `.venv/`) with deps
installed. Secrets go in `.env.local` (overrides `.env`, git-ignored).

## How it came to be (decisions made with the user)

1. **Rewrite language** â†’ stay Python, Flask â†’ FastAPI. New service only for the
   estimate, named `estimator-api`.
2. **Prompt source** â†’ local text file (`estimator_prompt.txt`), not Google Docs.
3. The prompt is purely a **renovation detector**: matches photos to a supplied
   `renovationItems` dataset, prices each as `FinalCost = DefaultRate Ă— Quantity`,
   returns `Totals.TotalRenovation`, `Summary Description`, `Guarantee`.
4. **Decisions on that prompt:** **single number** (no Normal/High split);
   **no renovation cap**.
5. **Output** â†’ return **JSON only** (HTML templates + Jinja2 removed).
6. **Removed OCC/BCI entirely** â€” the prompt never uses construction cost. So
   `bci.json`, postcode premium, 1986 cutoff, 40-year deduction, the guarantee
   band, `/initial-estimate`, and all OCC request fields are gone.
7. **Photos come from `rpId` only** (confirmed 2026-05-30). The service fetches
   them from calc.duo.tax. The old `photos` request field was removed.
8. **Model = `gpt-5.4-mini`** (vision reasoning model; chosen 2026-05-30).
   `openai` SDK bumped 1.59.6 â†’ 2.38.0. The vision call adapts by model family:
   reasoning models (gpt-5.x / o-series) send `reasoning_effort=low` +
   `max_completion_tokens` and NO temperature; classic models send
   `temperature=0`. Still Chat Completions, still `response_format=json_object`.
9. **Renovation items come from megamind** (added 2026-05-30). The old
   `renovationItems` request field was **removed**; the service fetches the
   catalog from megamind on **every** estimate. So the caller now sends only
   `rpId` (+ optional config/property/model).

### Request
```json
{
  "rpId": "48819125",                          // REQUIRED â€” the only required field
  "config": { "exclusions": ["landscaping"] }, // optional, forwarded to model
  "property": { "propertyType": "House" },     // optional context (soft hint only)
  "model": "gpt-5.4-mini"                       // optional override
}
```

### Response (what the model gives, currency-formatted)
```json
{
  "Renovations": [ { "_id","Name","Quantity","Unit","DefaultRate","FinalCost","Year" } ],
  "Renovations Total": "$200.00",
  "Summary Description": "...",
  "Disclaimer": "..."   // the model's "Guarantee" disclaimer, renamed
}
```
`_id` is the megamind item id, so results map back to the catalog.

### Errors
- `422` â€” bad/missing rpId, OR the rp_id yields no usable photos.
- `502` â€” megamind OR calc.duo.tax unreachable/bad response, OR the model failed.

### Renovation items via megamind  (built 2026-05-30; NOT yet live-tested)
- Endpoint: `GET {MEGAMIND_API_URL}/api/external/estimator-items` (base URL is
  the `MEGAMIND_API_URL` env var, default `https://api.megamind.duo.tax`; the
  path is appended in megamind_client). Header `X-API-KEY: <MEGAMIND_API_KEY>`.
- Returns a **bare JSON array** of catalog items (`id`, `name`, `defaultRate`,
  `unit`, plus `sections`/audit/`isDeleted`/`isVerified` fields).
- `_map_items` maps `id -> _id`, keeps `{_id, name, defaultRate, unit}`, drops
  `isDeleted` items and any missing id/name/rate. To restrict to vetted items,
  also skip `not item.get("isVerified")` (one line in megamind_client.py).
- Fetched fresh on every estimate (user wanted no caching).
- `python scripts/check_items.py` prints the mapped catalog (needs the key).

### Photos via rpId  (CONFIRMED + smoke-tested live 2026-05-30)
- Endpoint: `GET https://calc.duo.tax/property/{rp_id}/photos`, **no auth**.
- Bare JSON array of CoreLogic assets. `_map_photos` keeps Image / not
  noPhotoAvailable, drops `maps.googleapis.com` shots, picks
  `largePhotoUrl â†’ mediumPhotoUrl â†’ basePhotoUrl`, uses `scanDate`, sorts
  newest-first. Capped at 60 (`MAX_PHOTOS`).
- Verified vs rp_id 48819125: 28 raw â†’ 25 usable. `scripts/check_photos.py`.
